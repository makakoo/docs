title: "makakoo plugin: install and manage Makakoo plugins"
description: "Full reference for makakoo plugin subcommands — install, uninstall, enable, disable, update, outdated, start, stop, status, restart, and sync."
---

Plugins are the extension unit of Makakoo OS. Each plugin is a directory with a `plugin.toml` manifest and an optional entrypoint script. Plugins contribute SANCHO tasks, MCP tools, SKILL.md fragments embedded into CLI hosts, agent definitions, and infect fragments. The plugin registry lives at `$MAKAKOO_HOME/plugins/`; the lock file is at `$MAKAKOO_HOME/plugins.lock`.

Install sources come in three shapes: a local directory path, a `git+<url>[@<ref>]` URL pinned to a semver tag or 40-char SHA, or an `https://` tarball. Core-shipped plugins can be referenced by bare name using the `--core` flag.

---

## Subcommand overview

| Subcommand | Purpose |
|---|---|
| `plugin list` | All installed plugins with version and blake3 hash. |
| `plugin info <name>` | Parsed manifest and lock entry for one plugin. |
| `plugin install <source>` | Install a plugin from a local path, git URL, or tarball. |
| `plugin uninstall <name>` | Remove a plugin. `--purge` also wipes the state directory. |
| `plugin enable <name>` | Re-enable a previously disabled plugin. |
| `plugin disable <name>` | Soft-disable without uninstalling. |
| `plugin update <name>` | Re-fetch and reinstall from the recorded source. |
| `plugin outdated` | List plugins whose upstream ref has drifted (dry-run). |
| `plugin start <name>` | Start a service-kind or agent-kind plugin. |
| `plugin stop <name>` | Stop a running plugin. |
| `plugin status <name>` | Probe a plugin's health endpoint. |
| `plugin restart <name>` | Stop then start a plugin. |
| `plugin sync` | Batch-reinstall every plugin from `plugins-core/` into the live tree. |

## plugin install

Install a plugin from a local path, a git URL, or an HTTPS tarball.

```bash
makakoo plugin install <source> [--core] [--blake3 <hash>] [--sha256 <hash>] [--allow-unstable-ref]
```

### Source shapes

| Shape | Example |
|---|---|
| Local directory | `path/to/dir` or `/absolute/path` |
| Git URL (pinned) | `git+https://github.com/acme/my-plugin@v1.2.0` |
| Git URL (40-char SHA) | `git+https://github.com/acme/my-plugin@a1b2c3d4...` |
| HTTPS tarball | `https://example.com/my-plugin.tar.gz` (requires `--sha256`) |
| Core plugin by name | `skill-brain-multi-source` with `--core` |

### Flags

<ParamField path="source" type="string" required>
  Plugin source. See the table above for accepted shapes.
</ParamField>

<ParamField path="--core" type="boolean" default="false">
  Resolve `source` as a bare name against `$MAKAKOO_PLUGINS_CORE` (or the `plugins-core/` directory under the current repo).
</ParamField>

<ParamField path="--blake3" type="string">
  Expected blake3 hash of the plugin source tree. Takes precedence over the value declared in the manifest. Useful for pinning a specific build.
</ParamField>

<ParamField path="--sha256" type="string">
  Expected SHA-256 hash of the tarball bytes. Required for `https://` tarball sources. Ignored for local path and git sources.
</ParamField>

<ParamField path="--allow-unstable-ref" type="boolean" default="false">
  Permit non-tag, non-SHA git refs such as branch names (`main`, `master`). By default, git refs must be a semver tag or a 40-character SHA to prevent silent drift when a branch moves.
</ParamField>

### Examples

<CodeGroup>

```bash Git tag (recommended)
makakoo plugin install git+https://github.com/acme/skill-research-arxiv@v1.2.0
```

```bash Core plugin
makakoo plugin install skill-brain-multi-source --core
```

```bash Local path
makakoo plugin install /path/to/my-plugin
```

```bash HTTPS tarball
makakoo plugin install https://example.com/my-plugin.tar.gz \
  --sha256 a1b2c3d4e5f6...
```

```bash Development branch (unstable)
makakoo plugin install git+https://github.com/acme/my-plugin@main \
  --allow-unstable-ref
```

</CodeGroup>

<Warning>
Branch names (`main`, `master`, any non-tag non-SHA ref) are rejected by default. Pass `--allow-unstable-ref` only for development workflows. Pin to a semver tag for any plugin you want reproduced reliably.
</Warning>

## plugin uninstall

```bash
makakoo plugin uninstall <name> [--purge]
```

<ParamField path="name" type="string" required>
  Plugin name as reported by `makakoo plugin list`.
</ParamField>

<ParamField path="--purge" type="boolean" default="false">
  Also wipe the plugin's state directory (`$MAKAKOO_HOME/state/<plugin-name>/`) in addition to removing the install directory. State directories hold persistent data — use `--purge` only when you want a clean removal with no residual data.
</ParamField>

## plugin enable

Re-enable a previously soft-disabled plugin. The plugin directory stays untouched; the `plugins.lock` entry flips `enabled = true` and the next daemon registry load picks it up.

```bash
makakoo plugin enable <name>
```

<ParamField path="name" type="string" required>
  Plugin name.
</ParamField>

<Note>
A daemon restart is needed to pick up enable/disable changes. Run `makakoo daemon uninstall && makakoo daemon install` after toggling a plugin.
</Note>

---

## plugin disable

Soft-disable a plugin without uninstalling it. SANCHO task registration, MCP tool exposure, and infect fragment emission all skip the plugin on the next registry load. Nothing on disk changes.

```bash
makakoo plugin disable <name>
```

<ParamField path="name" type="string" required>
  Plugin name.
</ParamField>

## plugin update

Re-fetch and reinstall a plugin from its recorded source. State directories are preserved (no implicit `--purge`). The plugin's enabled/disabled flag is preserved across the reinstall.

```bash
makakoo plugin update <name> [--all] [--yes]
makakoo plugin update --all [--yes]
```

<ParamField path="name" type="string">
  Plugin name. Required unless `--all` is set.
</ParamField>

<ParamField path="--all" type="boolean" default="false">
  Update every updatable (git-sourced) plugin. Per-plugin failures are logged and skipped; the batch continues.
</ParamField>

<ParamField path="--yes" type="boolean" default="false">
  Skip the manifest-drift re-trust prompt. Use only when you trust upstream unconditionally. Without this flag, capability changes prompt for confirmation.
</ParamField>

<Note>
Tarball-sourced plugins cannot be auto-updated. You will see a hint to reinstall manually with a fresh `--sha256`.
</Note>

  Emit JSON instead of the default human-readable table.

## plugin start / stop / restart / status

Start, stop, restart, or probe the health of a service-kind or agent-kind plugin.

```bash
makakoo plugin start   <name>
makakoo plugin stop    <name>
makakoo plugin restart <name>
makakoo plugin status  <name>
```

Service plugins are backgrounded; stdout and stderr are redirected to:
- **macOS:** `~/Library/Logs/makakoo/<plugin>.{out,err}.log`
- **Linux:** `~/.local/state/makakoo/log/<plugin>.{out,err}.log`

`plugin status` probes `[service].health_endpoint` (HTTP if a URL, otherwise a shell command) or falls back to `[entrypoint].health`. Agent plugins use the same fallback chain as `makakoo agent status`.

## plugin sync

Batch-reinstall every plugin from `plugins-core/` into `$MAKAKOO_HOME/plugins/`. Used after a bulk source migration when the live install tree is out of date with the source tree.

```bash
makakoo plugin sync [--dry-run] [--force]
```

<ParamField path="--dry-run" type="boolean" default="false">
  Only report what would be reinstalled — do not modify disk.
</ParamField>

<ParamField path="--force" type="boolean" default="false">
  Uninstall and reinstall plugins that already have a target directory. Without `--force`, sync skips occupied directories. With `--force`, each plugin is atomically uninstalled and reinstalled, preserving state directories (no purge). Use when upgrading from the old manifest-only install shape to self-contained plugins.
</ParamField>

### Example

```bash
# Preview what sync would change
makakoo plugin sync --dry-run

# Apply the sync
makakoo plugin sync --force
```

## Plugin kinds

| Kind | Description |
|---|---|
| `skill` | Exposes one or more SKILL.md fragments to infected CLI hosts. |
| `agent` | A service with `start` / `stop` / `health` lifecycle commands. |
| `lib` | Python helper library with no SANCHO tasks or MCP tools. |
| `wiki` | Documentation-only; contributes no runtime behavior. |
