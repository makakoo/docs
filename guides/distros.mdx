A distro is a named, version-pinned bundle of plugins designed for a particular use case. Instead of installing plugins one by one, you pick a distro and get a coherent set — with blake3-pinned versions for reproducibility. Distros compose: each one can declare `include` references to other distros, so `core` extends `minimal`, and `sebastian` extends `core`.

## Available distros

| Distro | Display name | Purpose |
|--------|-------------|---------|
| `minimal` | Makakoo Minimal | Kernel-only install. Native Rust tasks only. Zero Python. |
| `core` | Makakoo Core | Default for most users. Adds browser harness, LLM gateway, watchdogs, SANCHO tasks, and memory consolidation. |
| `sebastian` | Sebastian Edition | Sebastian's daily-driver dogfood stack. Adds skill migrations from harvey-os on top of core. |
| `creator` | Makakoo Creator Edition | For writers, streamers, and artists. Brain-centric authoring and publishing loops. |
| `trader` | Makakoo Trader Edition | For market-facing autonomous agents. Adds watchdog and observability plugins for 24/7 trading loops. |

## What each distro includes

<AccordionGroup>

  <Accordion title="minimal — kernel only">
    The bare-bones install. Runs the native Rust SANCHO tasks and nothing else. No Python, no Python plugins. Every other distro includes `minimal.toml`.

    ```toml
    [distro]
    name = "minimal"
    display_name = "Makakoo Minimal"
    version = "0.1.0"
    description = "Kernel-only install. Runs the native Rust tasks. Zero Python."

    [kernel]
    version = "^0.1"

    [plugins]
    # No plugins — just the kernel's native handlers.
    ```

    **Best for:** testing, disk-constrained environments, or as a base for a fully custom plugin set.
  </Accordion>

  <Accordion title="core — the default">

## Installing a distro

<Steps>

  <Step title="Preview what will be installed">
    Run `--dry-run` to see the full resolved plugin list before committing:

    ```bash
    makakoo distro install core --dry-run
    ```
  </Step>

  <Step title="Install the distro">
    ```bash
    makakoo distro install core
    ```

    The command resolves `include` references, installs every plugin in the effective list, and writes `plugins.lock`.
  </Step>

  <Step title="Skip the confirmation prompt in CI">
    Pass `--yes` to suppress the interactive confirmation:

    ```bash
    makakoo distro install creator --yes
    ```
  </Step>

</Steps>

## Saving your current setup

Snapshot your live plugin set into a portable distro TOML file. Every enabled plugin is pinned to its exact version and blake3 hash:

```bash
makakoo distro save my-stack
makakoo distro save my-stack --out ~/my-stack.toml   # explicit output path
makakoo distro save my-stack --force                  # overwrite if it exists
```

Replay it on another machine:

```bash
makakoo distro install --from ~/my-stack.toml --yes
```

<Tip>
  Disabled plugins are excluded from `distro save` by default so the distro replays only what was active. Add `--include-disabled` to capture everything on disk.
</Tip>

## Distro composability

Every distro file can declare `include` references to other distros. Makakoo resolves the full dependency graph before installing. For example, `sebastian.toml` includes `core.toml`, which includes `minimal.toml` — installing `sebastian` brings in every plugin from all three:

```toml
include = ["core.toml"]
```

Run `makakoo distro install <name> --dry-run` at any time to see the fully resolved flat plugin list before committing.
