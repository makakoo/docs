Makakoo OS ships a manifest-driven plugin system where every extension â€” agents, skills, scheduled tasks, monitors, patterns, and mascots â€” is a directory containing a `plugin.toml` that declares its identity, source, required capabilities, and lifecycle entrypoints. The core runtime enforces those declarations at call time via a Unix socket capability sandbox, so even a compromised plugin can only access what its manifest grants.

## How the plugin system works

Each plugin runs as an isolated process. When the plugin calls a capability (read Brain, call an LLM, make an HTTP request), the request travels over a per-plugin Unix socket to the Makakoo core, which checks the declared grants and either allows or denies the call. Denied calls are written to the audit log at `~/.makakoo/logs/audit.jsonl`.

Plugins are written in Python or Rust. Python plugins use a self-contained `.venv` bootstrapped by `makakoo plugin internal venv-bootstrap` during install. Communication between your plugin code and Makakoo core uses the `makakoo` Python client or raw JSON-RPC over the socket.

## Plugin kinds

| Kind | Purpose | Examples |
| --- | --- | --- |
| `agent` | Long-running autonomous workers | `agent-harveychat`, `agent-arbitrage-agent` |
| `skill` | Reusable AI workflows and prompt packages | `skill-research-arxiv`, `skill-dev-ship` |

## Basic plugin commands

List installed plugins and inspect a specific plugin:

```bash
# List all installed plugins
makakoo plugin list

# List with JSON output
makakoo plugin list --json

# Show parsed manifest + lock entry for a plugin
makakoo plugin info skill-research-arxiv
```

Install from the core library, a git URL, or a local directory:

```bash
# Install a core plugin by name
makakoo plugin install skill-research-arxiv --core

# Install a pinned git release
makakoo plugin install git+https://github.com/user/my-plugin@v1.0.0

# Install from a local directory
makakoo plugin install ./my-plugin
```

Enable, disable, update, and remove:

```bash
makakoo plugin disable skill-research-arxiv
makakoo plugin enable  skill-research-arxiv
makakoo plugin update  skill-research-arxiv
makakoo plugin uninstall skill-research-arxiv
```

Check for available updates without touching disk:

```bash
makakoo plugin outdated
makakoo plugin update --all
```

## Capability grants

Every plugin declares exactly which capabilities it needs in `plugin.toml`. The runtime denies anything not listed.

```toml
[capabilities]
grants = [
  "brain/read",                               # read journal entries
  "brain/write",                              # write to journals
  "llm/chat",                                 # call the LLM gateway
  "net/http:https://clob.polymarket.com/*",   # HTTP to one domain
  "secrets/read:POLYMARKET_API_KEY",          # one named secret
  "state/plugin",                             # plugin's own state dir
]
```

Available capability verbs:

| Verb | Scope parameter | Purpose |
| --- | --- | --- |
| `brain/read` | â€” | Read Brain journals and pages |
| `brain/write` | â€” | Write to Brain journals |
| `llm/chat` | model (optional) | Call the LLM gateway for chat |
| `llm/embed` | model (optional) | Request vector embeddings |
| `net/http` | URL glob | Outbound HTTP to matching URLs |
| `secrets/read` | key name | Read one named secret from keyring |
| `state/plugin` | â€” | Read/write own state directory |
| `fs/read` | path glob | Read files matching a path pattern |
| `fs/write` | path glob | Write files matching a path pattern |
